# Архитектура и «агенты» проекта

Документ описывает ключевые составляющие бота (условные «агенты») и их роли, а также основные потоки данных при обработке запросов пользователей.

## Обзор компонентов

- Бот (aiogram) — `src/bot/bot.py`
  - Инициализация окружения (`dotenv`), создание `Bot` и `Dispatcher`.
  - Подключение маршрутизатора из `handlers.py`.
  - Запуск `start_polling`.

- Роутер и обработчики — `src/bot/handlers.py`
  - Регистрация команд `/start`, `/help`.
  - Прием входящих сообщений/документов и вывод контекстных подсказок.
  - FSM (машина состояний) `ConvertStates` для пошагового выбора задачи и параметров:
    - выбор типа задачи (PDF→JPEG, JPEG→WEBP/AVIF/ICO, JPEG→PDF)
    - выбор `quality` (для JPEG/WEBP/AVIF)
    - выбор `dpi` (для PDF→JPEG)
    - выбор режима PDF (единый документ или пофайлово)
    - ввод пользовательских размеров для ICO
  - Формирование inline-клавиатур для выбора параметров.
  - Вызов функций из слоя конвертации и отправка результата пользователю.

- Слой конвертации — `src/core/image_converter.py`
  - Набор функций для конвертации и вспомогательных операций:
    - `convert_pdf_to_jpegs`, `convert_jpeg_to_webp`, `convert_jpeg_to_avif`, `convert_jpeg_to_ico`, `convert_jpegs_to_pdf`.
    - `handle_zip_input` — безопасная распаковка, рекурсивный сбор входных файлов, пакетная конвертация, упаковка результатов.
    - Сервисные функции: безопасная распаковка ZIP, форматирование размера, создание выходных директорий, архивация и т.д.
  - Точки входа верхнего уровня (узко-назначенные «агенты» задач):
    - `process_pdf_to_jpeg(input_path, quality, dpi)`
    - `process_jpeg_to_webp(input_path, quality)`
    - `process_jpeg_to_avif(input_path, quality)`
    - `process_jpeg_to_ico(input_path, sizes)`
    - `process_jpeg_to_pdf(input_path, pdf_mode)`
  - Валидация зависимостей по требованию (ленивая загрузка Pillow, PyMuPDF, img2pdf).

- Вспомогательный модуль — `src/core/converter.py`
  - Интерфейсный слой/обертки для вызова конвертеров из `image_converter.py` и возможная дальнейшая декомпозиция.

- Конфигурация — `src/bot/config.py`, `src/core/config.py`
  - Чтение `BOT_TOKEN` из окружения через `dotenv`.

## Потоки данных

1. Пользователь отправляет файлы боту (документ/изображение/ZIP).
2. Обработчик в `handlers.py` определяет тип входа и предлагает релевантные задачи.
3. Пользователь выбирает задачу и параметры через inline-кнопки или ввод.
4. FSM накапливает контекст (выбранные опции).
5. Хэндлер запускает соответствующую функцию «агента» из `image_converter.py`.
6. Результат(ы) сохраняются во временную директорию, при необходимости упаковываются в ZIP.
7. Бот отправляет файлы пользователю и очищает временные данные.

## Состояния и параметры (FSM)
- `ConvertStates.task` — тип задачи.
- `ConvertStates.quality` — качество (для JPEG/WEBP/AVIF).
- `ConvertStates.dpi` — DPI для PDF→JPEG.
- `ConvertStates.pdf_mode` — режим сборки PDF из JPEG(ов).
- `ConvertStates.ico_sizes` — набор размеров для ICO.

## Обработка ZIP
- Безопасная распаковка: проверка путей, предотвращение directory traversal.
- Сбор входных файлов только нужных расширений.
- Конвертация в целевом режиме (с учетом выбранных параметров).
- Сбор/упаковка результатов при необходимости обратно в ZIP для удобной выдачи пользователю.

## Логи и диагностика
- Базовая конфигурация логирования в `bot/bot.py` (`logging.INFO`).
- Ошибки при зависимостях/конвертации поднимаются как исключения, перехватываются в обработчиках и отражаются пользователю дружелюбными сообщениями.

## Расширение и масштабирование
- Добавление новых задач:
  - Расширить клавиатуру/состояния в `handlers.py`.
  - Реализовать новую функцию верхнего уровня в `image_converter.py` и/или переиспользовать существующие примитивы конвертации.
- Горизонтальное масштабирование: можно запускать несколько экземпляров контейнера бота за счет статeless-природы; следите за лимитами Telegram Bot API.
- Хранилище/кэш: при необходимости можно добавить внешнее хранилище для больших артефактов; сейчас бот работает с локальными временными файлами.
