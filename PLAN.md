# План: Telegram-бот для конвертации изображений

## Текущее состояние

Уже есть `image_converter.py` — CLI-утилита со всей логикой конвертации:
- PDF → JPEG (с настройкой quality и dpi)
- JPEG → PDF (combine / per-file)
- JPEG → ICO (с настройкой размеров)
- JPEG → WebP (с настройкой quality)
- Поддержка ZIP-архивов на входе и выходе

---

## Архитектура бота

```
pyproject.toml      — метаданные проекта, зависимости (uv)
.python-version     — закреплённая версия Python
bot.py              — точка входа, запуск бота
handlers.py         — обработчики команд и сообщений
converter.py        — обёртка над image_converter.py для бота
config.py           — токен, лимиты, дефолтные параметры
.env                — BOT_TOKEN (не коммитится, в .gitignore)
```

Менеджер пакетов: **uv** — lockfile (`uv.lock`), виртуальное окружение (`.venv`), запуск через `uv run`.

Библиотека: **aiogram 3.x** (asyncio, роутеры, FSM из коробки).

## Шаги реализации

### Шаг 1. Настройка проекта (uv)

- [ ] `uv init` — инициализировать проект, создаст `pyproject.toml` и `.python-version`
- [ ] `uv add aiogram Pillow pymupdf img2pdf` — добавить зависимости (создаст `uv.lock` и `.venv`)
- [ ] Создать `.env` с `BOT_TOKEN=...` и добавить в `.gitignore`
- [ ] Создать `config.py` — читает `BOT_TOKEN` из env, лимиты размеров файлов, значения по умолчанию (quality=85, dpi=150, ico_sizes)

### Шаг 2. Конвертер-обёртка (`converter.py`)

- [ ] Написать async-функцию `convert(file_path, task, options) -> Path` — принимает путь к скачанному файлу, тип задачи и параметры, возвращает путь к результату
- [ ] Внутри: создаёт `tempfile.TemporaryDirectory`, вызывает нужную функцию из `image_converter.py`, если результат — директория, пакует в ZIP
- [ ] Обработка ошибок с понятными сообщениями на русском

### Шаг 3. Обработчики (`handlers.py`)

#### 3.1 Команда `/start`
- [ ] Приветственное сообщение с описанием возможностей бота
- [ ] Список поддерживаемых конвертаций

#### 3.2 Команда `/help`
- [ ] Подробная инструкция: как отправить файл, как выбрать формат, как указать quality/dpi

#### 3.3 Приём файла (document handler)

Сценарий работы:
1. Пользователь отправляет файл (PDF, JPEG, ZIP)
2. Бот определяет тип файла по расширению
3. Бот предлагает inline-кнопки с доступными конвертациями:
   - PDF файл → кнопка `PDF → JPEG`
   - JPEG файл → кнопки `JPEG → PDF`, `JPEG → ICO`, `JPEG → WebP`
   - ZIP файл → все 4 кнопки (бот не знает содержимое заранее)
4. Пользователь нажимает кнопку
5. Бот спрашивает параметры (или использует дефолтные):
   - Для PDF→JPEG: quality (1-100) и dpi (72/150/300) — inline-кнопки с пресетами + "по умолчанию"
   - Для JPEG→WebP: quality (1-100) — inline-кнопки с пресетами
   - Для JPEG→ICO: размеры — дефолтный набор или custom
   - Для JPEG→PDF: режим combine/per-file (если в ZIP несколько файлов)
6. Бот скачивает файл, конвертирует, отправляет результат
7. Бот удаляет временные файлы

- [ ] Реализовать document handler — сохраняет file_id, определяет тип, показывает кнопки
- [ ] Реализовать callback query handler — обработка нажатий кнопок
- [ ] Реализовать FSM (aiogram StatesGroup) для пошагового выбора параметров

#### 3.4 Приём фото (photo handler)
- [ ] Если пользователь отправляет фото (не как документ) — предупредить, что Telegram сжимает фото, и попросить отправить как файл (скрепка → файл)

### Шаг 4. Точка входа (`bot.py`)

- [ ] Создание `Bot` + `Dispatcher`, подключение роутера из `handlers.py`
- [ ] Запуск polling (`dp.start_polling`)
- [ ] Graceful shutdown
- [ ] Запуск: `uv run python bot.py`

### Шаг 5. Ограничения и валидация

- [ ] Лимит размера файла — Telegram API даёт скачивать до 20 МБ, отправлять до 50 МБ. Предупреждать пользователя при превышении
- [ ] Таймаут на конвертацию — если файл слишком большой / PDF на 500 страниц, не зависать бесконечно
- [ ] Валидация: проверять что в ZIP есть нужные файлы, что JPEG валидный и т.д.
- [ ] Очистка temp-файлов даже при ошибках

### Шаг 6. UX-мелочи

- [ ] Отправлять "typing" / "upload_document" action пока идёт конвертация
- [ ] Показывать размер файла до и после конвертации
- [ ] Если результат — один файл, отправлять файлом; если много — ZIP-архивом
- [ ] Логирование ошибок в консоль

## Пример пользовательского сценария

```
Пользователь: /start
Бот: "Привет! Я конвертирую файлы. Отправь мне PDF, JPEG или ZIP-архив."

Пользователь: [отправляет photo.jpg как документ]
Бот: "Получен JPEG файл. Что сделать?"
     [JPEG → PDF]  [JPEG → ICO]  [JPEG → WebP]

Пользователь: [нажимает JPEG → WebP]
Бот: "Выберите качество:"
     [50 — мин. размер]  [75]  [85 — по умолчанию]  [95 — макс. качество]

Пользователь: [нажимает 85]
Бот: "Конвертирую..."
Бот: [отправляет photo.webp]
     "Готово! 2.3 MB → 0.8 MB"
```
