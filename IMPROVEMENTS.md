#### 1) UX и сценарии пользователя
- Авто-детект задач по файлу и «быстрые пресеты»
  - Сейчас клавиатура строится от расширения (`build_task_keyboard`). Добавить «Быстрый старт»: при doc/photo сразу запускать дефолтную операцию с возможностью «Отменить/Изменить параметры».
- Прогресс и статус
  - Во время долгих задач отправлять «⌛ Выполняю…» + периодические апдейты (editMessageText), по таймауту — вежливое объяснение и советы снизить `dpi/quality`.
- Инлайн‑кнопка «Повторить с другими параметрами»
  - После результата предложить повтор с другим `quality/dpi/pdf_mode/ico_sizes` — подставляя предыдущий ввод из FSM.
- Предпросмотр и размер результата
  - Для изображений: миниатюра + `format_size()` результата до скачивания, чтобы пользователь понимал выгоду.
- Валидация ввода
  - Для `ico_custom_sizes` — подсветка ошибок и подсказки (например, допустимые: 16, 32, 48, 64, 128, 256).

#### 2) Поддерживаемые форматы и опции конвертации
- Больше форматов
  - Вход: `PNG/TIFF/BMP/HEIC/SVG` → JPEG/WebP/AVIF/PNG. Для HEIC — `pillow-heif`; для SVG → PNG — `cairosvg`/`rsvg-convert`.
- PDF улучшения
  - Разделение PDF по страницам и «диапазон страниц» (1–N).
  - Объединение нескольких PDF (режим «merge»). Сейчас есть `jpeg-to-pdf` (combine/per-file) — расширить на PDF↔PDF.
- Оптимизация изображений
  - Авто‑ориентация (EXIF), профиль sRGB, настройка `subsampling`, опция «без EXIF» (приватность + меньше вес).
  - «Цель по размеру файла» (подбор качества до X МБ).
- Водяные знаки/оверлеи и ресайз
  - Пресеты: «Аватарки», «Иконки», «Превью для web» с целевыми сторонами.

#### 3) Производительность и масштабируемость
- Очередь задач
  - Выделить фоновые воркеры (RQ/Celery/Arq) и Redis для задач. Сейчас `convert()` уже выносит CPU в `asyncio.to_thread` — следующий шаг: очередь + пул воркеров.
- Параллелизм на ZIP/многих страницах
  - В `handle_zip_input` и `convert_pdf_to_jpegs` распараллелить страницы/файлы с лимитом по CPU/памяти.
- Стриминг I/O
  - Избегать лишних копирований; по возможности стримить скачивание/загрузку, работать в temp с mmap/списками файлов.
- Кэширование результатов
  - Хэш входа + опций → кэш в файловой системе/S3. Повторные запросы — мгновенно.

#### 4) Надёжность, лимиты и операционка
- Границы и защита ресурсов
  - Лимиты уже есть в `settings` (`max_download_mb`, `convert_timeout_seconds`). Добавить: ограничение страниц PDF, пикселей/измерений, fail‑fast при слишком больших картинках.
- Безопасность архивов
  - У вас есть `safe_extract_zip` — отлично. Добавить антивирусную проверку (опционально, например, ClamAV сервис) и фильтрацию типов внутри ZIP.
- Повторы/ретраи
  - Обернуть внешние операции (загрузка файла, отправка результата) в ретраи с backoff.
- Логи и трассировка
  - Структурные логи (JSON), кореляционные ID на задачу, сохранение ключевых метрик: формат, размеры входа/выхода, время, ошибки.
- Метрики/алерты
  - Экспорт в Prometheus/OpenTelemetry: RPS, время конвертации, ошибки по типам, топ‑параметры.

#### 5) Безопасность и приватность
- Белый список уже есть (`AccessMiddleware`). Добавить:
  - Rate‑limit по пользователю, анти‑флуд, защита от «штормов» при покрытии очередью.
  - Маскирование чувствительных полей в логах (токены, user id при необходимости).
  - Опция авто‑удаления временных файлов после N минут (плюс «Удалить мои данные» командой).

#### 6) DX (Developer Experience) и тесты
- Тестовое покрытие сценариев
  - В `tests/test_conversions.py` добавить кейсы: большие PDF, битые изображения, пустые ZIP, custom ICO sizes, разные `pdf_mode`.
- Контроль качества изображений
  - Сверять размеры/форматы/метаданные результата, не только существование файлов.
- Локальная разработка
  - Makefile уже есть — добавить команды: lint (`ruff`/`flake8`), type‑check (`mypy`), test‑watch, форматирование (`black/ruff format`).
- CI/CD
  - GitHub Actions/GitLab CI: линтеры, тесты, сборка Docker. Push в регистри, релизы с ченджлогом.

#### 7) Docker и деплой
- Лёгкий образ
  - Мульти‑стейдж, slim‑база, `Pillow`/`pymupdf`/`img2pdf` системные зависимости только необходимые, включить healthcheck.
- Конфигурация
  - Больше параметров в `Settings`: лимиты страниц, поведение кэша, включение/выключение форматов и тяжёлых функций (HEIC/SVG), флаги приватности (удалять EXIF).
