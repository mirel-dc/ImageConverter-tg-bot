# Image Converter Bot

Telegram-бот для конвертации изображений и PDF. Поддерживает:
- PDF → набор JPEG (с выбором DPI и качества)
- JPEG → WEBP
- JPEG → AVIF
- JPEG → ICO (стандартные и пользовательские размеры)
- JPEG(ы) → PDF (режимы: один PDF на все изображения или отдельный PDF на каждый файл)
- Обработку ZIP-архивов с пакетной конвертацией и формированием результатов (в т.ч. ZIP на выходе при необходимости)

Основан на:
- aiogram 3
- Pillow (+ AVIF плагин)
- PyMuPDF
- img2pdf
- python-dotenv
- uv (управление зависимостями и запуск)

## Быстрый старт

1) Создайте файл `.env` в корне проекта:
```
BOT_TOKEN=ваш_токен_бота
```
Как получить токен: у @BotFather в Telegram.

2) Запуск одним из способов ниже.

### Вариант A — локально (uv)
Требования: Python 3.11+, установленный `uv`.

Команды:
```
cd src
uv run python -m bot.bot
```

Тесты:
```
cd src
uv run pytest ../tests/
```

Для E2E тестирования сложных сценариев (например, ZIP-архивы с вложенными папками или смешанным контентом) используйте:
```
cd src
uv run pytest ../tests/test_e2e_scenarios.py
```

### Вариант B — Docker Compose
Требования: Docker и Docker Compose.

Команды:
```
# сборка и запуск в фоне
docker compose up -d --build

# просмотреть логи
docker compose logs -f

# остановить и удалить контейнеры
docker compose down
```

Файл `.env` из корня автоматически прокидывается внутрь контейнера.

### Вариант C — Makefile (если доступна команда make)
Доступные цели:
```
make up      # docker compose up -d --build
make logs    # docker compose logs -f
make down    # docker compose down
make run     # локальный запуск через uv (cd src && uv run python -m bot.bot)
make test    # локальный запуск тестов через uv
```

## Использование бота
- Отправьте боту файл (PDF, JPEG или ZIP).
- Бот предложит выбрать задачу (тип конвертации) и параметры (качество, DPI, режим PDF, размеры ICO).
- Дождитесь результата: бот вернет файл(ы) или архив с результатами.

## Основные возможности и параметры
- PDF → JPEG: настройка `dpi` и `quality` (качество JPEG).
- JPEG → WEBP/AVIF: настройка `quality`.
- JPEG → ICO: выбор стандартных размеров или ввод собственных (через запятую, например `16,32,64,128`).
- JPEG(ы) → PDF:
  - «один PDF на все изображения» (коллекция в один документ)
  - «отдельный PDF на каждый файл»
- ZIP-вход: безопасная распаковка, рекурсивный сбор входных файлов соответствующего типа, конвертация и упаковка результатов (при необходимости) обратно в ZIP.

## Разработка
- Минимальная версия Python: 3.11
- Стиль/запуск: проект ориентирован на `uv` для воспроизводимости окружения:
```
cd src
uv run python -m bot.bot
```
- Тесты: `pytest` (запуск см. выше)
- Логи: настраиваются через `logging.basicConfig(level=logging.INFO)` в `bot/bot.py`.
